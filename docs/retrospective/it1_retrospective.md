The biggest takeaway from Iteration 1 is that we need to stop getting caught in decision paralysis during planning. Trying to figure out every tiny detail before implementing is not the most productive. Only after we started implementing could we really think about finalizing our stack and making the best decision for our project.

Moving forward, we should aim to start our dev work earlier in order to allow time for last minute issues that might arise. We were lucky that our late start didnt cause any issues with the functionality of our code at the deadline but for future reference, now that we understand the time line of how long things take, we should start sooner.

In terms of what we implemented, we proved the core of our app actually works. We’ve got data fetching, storage, and action logic all working together. Knowing we can handle the full process from an API endpoint to a frontend display is a huge win. Now we want to focus on making the UI better and deciding exactly how we want users to interact with the platform. This might include adding more visuals/data representations, adding leaderboard functionalities once we decide on the final direction of what kind of data we want to use and how we want the users to interact with our app.
